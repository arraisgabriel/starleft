## EPIC B — Relationship Ledger systems

> The Relationship Ledger is the one genuinely *new* persistent state in The Off-Hours (design §5). This epic builds the bond data-model, its no-cold-start seeding, the deterministic compatibility function that biases vet↔vet trends, the tier-up → milestone-scene trigger, and the gentle decay/closest-nemesis layer. It assumes Epic A has already created `CAMPAIGN.offhours = {v, nights, bonds:{}, visited:{}}` with legacy-safe defaults (§5.1, the A2 worked example). Every task here lives in a new runtime file `js/offhours.js` (loaded after `npc_lore.js`, before `ui.js` — §12.1) and reads/writes only that defaulted state, so an old save still loads with zero Off-Hours data. All randomness routes through `makeRng` (js/state.js:25) for netRole-byte-stability; all bond *mutations* are host-authoritative (committed via the Epic-A net wrapper), while bond *reads* (tiers, names, decay cues, closest flags) are pure/local and safe on a client.

It is **pure local UI** built on the existing `openHubMenu(spec)` shell (the same shell the Wake/Training/Condo menus ride), so it works identically on solo/host/**client** (clients open, read, and scroll bonds locally; *committing* an outcome is a separate net path owned by the conversation engine, not this epic). Every task here renders state and routes button clicks into the engine wrappers authored elsewhere — it never simulates a bond mutation itself. The discipline mirrors `openWakeMenu`/`buildWakeBody`/`wakeSignature` (js/ui.js:1417-1539): a `signature()`-gated rebuild so choice buttons are never eaten mid-click, `hubMenuColumns(2)` desktop / single-column mobile, and `hubMenuUnitCard` for every portrait. Design refs: §4.1, §6.4, §12.1, §12.3.

## EPIC G — Systemic backdrop — the blend

The thin, transient mood floor that keeps the city breathing between the authored milestone scenes (§7). It does only four things: (G1) derive a per-vet `u.mood` that is recomputable and never identity, (G2) surface a two-directional "needs a night out" prompt list so the floor becomes actionable opportunities (both a vet *wants* something and an NPC *asks for* a vet), (G3) let the world stage its own vet↔vet / vet↔NPC moments as deterministic, closed-form speech-bubble ambient with a tiny self-tick on witnessed bonds, and (G4) hang a seek-out / "caught" prompt over an already-happening moment that the player can convert into a staged directed scene. Everything here is **light**: the hard line in §7.4 is that this is a thin reactive floor, not a Sim — if any of it starts needing balance passes, it gets cut back to caught-moments only. All of it is derived/transient or append-only, holds across all three netRole paths (mood is read-only on the client; the only mutations — the self-tick bond bump and a converted seek-out scene — route host-authoritative), and loads a pre-feature save with zero Off-Hours data.

**Phase:** P2 (depends on the Epic A Ledger + the Epic C/E venue + bond machinery; this epic is the systemic layer that sits on top of them).
