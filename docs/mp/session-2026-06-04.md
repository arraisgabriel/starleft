## TL;DR — where the system stands

- Two coexisting netcode paths, switched by the master flag **`window.USE_ROLLBACK`** (default **false**):
  - **Host-authoritative** (ships today, default): `sync.js` + `commands.js`. Host simulates & broadcasts
    snapshots; client applies + interpolates + computes local fog. Phases 0–4 + robustness landed earlier.
  - **Symmetric rollback** (new, behind the flag): `rollback-*.js` over the Trystero transport. Both peers
    simulate; the `rollback-netcode` library handles prediction, rollback, StateSync on join, and desync
    detection. Config: `Star` topology, `Host` desync authority, `tickRate:60`, `hashInterval:30`, `maxPlayers:3`.
- **Determinism is the foundation** and is proven: seeded LCG RNG `simRandom`/`seedSim` in
  [js/state.js](../../js/state.js) (cursor `_simRngS`, serialized with state). Cross-engine identical
  (Chrome V8 == Safari JSC, signature `0x35d253c3`).
- **New this session:** an in-game **MP debug panel** + ~53 `NET.mpLog` call sites, and a repeatable
  **Playwright test** (`docs/mp/playwright/`). The rollback live path is now **machine-validated** (handshake,
  no-DESYNC, command convergence, disconnect surfaced) — see results below.
- **3-player:** the rollback config allows `maxPlayers:3`, but the **lobby is still 2-player** (Phase C seats
  not built). Rollback **drop recovery** is also Phase C (a disconnect is *logged*, not yet *recovered*).

  `.mp-dbg-row.mp-dbg-{info|ok|warn|err}`. Loaded early in `rts.html` so `NET.mpLog` exists before any handler.

## How to debug / reproduce going forward

**Serve (required for WebRTC; file:// disables it):**
```bash
python3 -m http.server 8000          # from repo root → http://localhost:8000/rts.html
```

**Console one-liners (open the page, then in DevTools):**
```js
NET.determinismTest(0, 900, 12345)        // intra-engine determinism → "[det] PASS"
NET.determinismSig(0, 5400, 12345)        // cross-browser: run in 2 browsers, compare the 0x… signature
NET.rbRoundTripTest(0, 600, 600, 12345)   // serialize completeness → "[rb] ROUND-TRIP PASS"
NET.rbLocalTest(0, 600, 12345)            // 2 in-process Sessions + StateSync → "[rb] LOCAL PASS" (async)
NET.rbLocalCmdTest(0, 600, 12345)         // input model + real rollback → "[rb] CMD PASS" (async)
```

**Live rollback co-op (manual):** open two browser windows; in BOTH set `window.USE_ROLLBACK = true`
**before** the host clicks Start (the client's main loop only drives `rbStepLoop` when its own flag is true);
host Create Room, client Join by code, host Start. Open `🐞 Debug` on each to watch the handshake; `Copy`
yields a paste-ready report.

**Automated:**
```bash
bash docs/mp/playwright/run-mp-test.sh    # full 3-tier; writes docs/mp/playwright/report.md
```

### Playwright gotchas discovered (baked into the driver — reuse these)
- A **boot gate** (`#bootGate`, audio/start) overlays the menu on load — dismiss it via a real click on
  `#btn-simulate` before any lobby interaction, or clicks won't land.
- `G`, `running`, `netRole` are **`let`-scoped globals**, *not* `window` properties → eval the **bare
  identifier** (`G.entities`, `running`), never `window.G`.
- `window.MP_SESSION` and `window.MP_UI` **are** real window objects → read the room code via
  `window.MP_SESSION.code`.
- `playwright-cli eval` of a **multi-statement** string truncates to the first statement → set each thing in
  its own eval, or wrap logic in an IIFE `(function(){ … })()`.
- `--raw eval` of a **string/JSON return** comes back quoted with escaped `\"` → strip outer quotes; for
  comparisons prefer `.join(',')` over `JSON.stringify` to avoid escaping.
- Tier 3 needs **outbound Nostr relays** (signaling) — the driver probes `relay.damus.io` and SKIPs (not
  fails) Tier 3 if unreachable.

## Quick triage when live co-op "doesn't work"
Open `🐞 Debug` on **both** peers and read/Copy the log. Typical signatures:
- No `peer connected` / `host welcomed us` within ~20s → signaling/NAT (paste a TURN relay in the lobby).
- `join timed out` / `sync timed out` → bad code or host snapshot never arrived.
- `DESYNC @tick … local 0x… vs remote 0x…` (rollback) → a non-deterministic sim path; reproduce with
  `rbRoundTripTest`/`rbLocalCmdTest` and check for `Math.random()`/`Date.now()` or transcendental drift.
- `watchdog: no host snapshot …` (host-auth) → host crashed/dropped.
- `rollback transport: peer … disconnected` / `host link dropped` → expected on a real drop (recovery is Phase C).
